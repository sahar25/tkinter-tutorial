What are Widgets?

A widget is just a word for any visual piece of a window — a button, a text box, a label. Everything you can see and click in a GUI (Graphical User Interface) is a widget.

Here are the 10 most common tkinter widgets. You'll learn all of them in this tutorial:

Label
Display text or images
Button
Trigger an action on click
Entry
Single-line text input
Listbox
Select from a list of items
Radiobutton
Choose ONE option from a set
Checkbutton
Toggle one or more options
Frame
Group other widgets together
Canvas
Draw shapes or place images
Scrollbar
Navigate large content areas
Combobox
Dropdown list with text entry
💡
How every widget works Every widget follows the same two steps: 1) Create it and tell it what it looks like. 2) Place it in the window so it appears on screen. That's it!

Your First Window

Start here

Before any widgets, we need a window to put them in. This is the skeleton that every tkinter program starts with. Copy it, run it, and you'll see a blank window appear!

 my_first_window.py 
# Step 1: Import tkinter and give it the short name "tk"
import tkinter as tk

# Step 2: Create the main window (we call it "root" by convention)
root = tk.Tk()

# Step 3: Give the window a title (shows in the title bar)
root.title("My First Window!")

# Step 4: Set the size — width x height in pixels
root.geometry("400x300")

# --- Add your widgets here (we'll get to this soon!) ---

# Step 5: Start the window — it stays open until the user closes it
root.mainloop()
Line of codeWhat it means in plain English
import tkinter as tkLoad the tkinter toolkit. We shorten it to "tk" so we don't have to type "tkinter" every time.
root = tk.Tk()Create the main window. We store it in a variable called "root" (you can call it anything you like).
root.title("...")Set the text that appears in the window's title bar at the top.
root.geometry("400x300")Set the window size: 400 pixels wide, 300 pixels tall.
root.mainloop()This is the "keep the window running" line. Without it, the window would flash open and close instantly. Always put this LAST.
What you'll see when you run it A blank window, 400×300 pixels, with "My First Window!" in the title bar. That's it — but now you have the foundation to add any widget!
⚠️
Important rule! root.mainloop() must ALWAYS be the very last line of your program. All your widgets and code go ABOVE it.

Label & Button

Widget

A Label shows text on the screen. A Button runs a function when the user clicks it. These are the two most used widgets in any GUI.

 label_and_button.py 
import tkinter as tk

# This function runs when the button is clicked
def button_clicked():
    # Change the label's text using .config()
    my_label.config(text="You clicked the button! 🎉")

# Create the main window
root = tk.Tk()
root.title("Label and Button")
root.geometry("350x200")

# Create a Label widget
# - root    = put it inside the main window
# - text    = what it says
# - font    = (font name, size)
my_label = tk.Label(root, text="Click the button below!", font=("Helvetica", 14))

# .pack() places the widget in the window (top to bottom, centred)
# pady adds space above and below (in pixels)
my_label.pack(pady=30)

# Create a Button widget
# - command = the function to run when clicked (no brackets!)
my_button = tk.Button(root, text="Click Me!", command=button_clicked)
my_button.pack()

root.mainloop()
Part of the codeWhat it does
def button_clicked():We define a function BEFORE creating the button, so the button can find it.
my_label.config(text="...").config() lets you change a widget's settings after it's been created. Here we change the text.
tk.Label(root, text="...", font=(...))Create a label. "root" means "put this inside the main window". We set the text and font.
.pack(pady=30)Place the widget. pady=30 adds 30 pixels of space above and below. Try changing this number!
command=button_clickedWhen clicked, run button_clicked. Notice: NO brackets after button_clicked — we're not calling it yet, just pointing to it.
🎯
Try it yourself! After copying and running the code, try: (1) change the font size from 14 to 20, (2) change "Click Me!" to your own button text, (3) add a second button that changes the label to something different.

Entry — Getting Text from the User

Widget

An Entry widget is a text box where the user can type. We use .get() to read whatever they typed.

 entry_input.py 
import tkinter as tk

def say_hello():
    # .get() reads the text currently in the Entry box
    name = name_entry.get()

    # Use the name to update the greeting label
    greeting_label.config(text=f"Hello, {name}! 👋")

root = tk.Tk()
root.title("Say Hello")
root.geometry("350x220")

# A label that tells the user what to type
tk.Label(root, text="Type your name:", font=("Helvetica", 12)).pack(pady=(30, 5))

# The Entry widget — where the user types
# width = how many characters wide it looks
name_entry = tk.Entry(root, width=25, font=("Helvetica", 12))
name_entry.pack()

# A button that calls say_hello when clicked
tk.Button(root, text="Say Hello!", command=say_hello).pack(pady=12)

# An empty label — we'll fill it in when the button is clicked
greeting_label = tk.Label(root, text="", font=("Helvetica", 14))
greeting_label.pack()

root.mainloop()
Part of the codeWhat it does
name_entry.get()Reads whatever the user has typed in the box and gives it back as a string.
tk.Label(root, text="", ...)Creating a label with empty text — a "placeholder" we'll fill in later with .config()
pady=(30, 5)Two numbers in brackets: first is space ABOVE, second is space BELOW. Handy for fine-tuning gaps.
f"Hello, {name}!"An f-string — you already know these from Python. We drop the variable right into the text.

Useful things you can do with Entry

CodeWhat it does
entry.get()Read what's typed — gives back a string
entry.delete(0, tk.END)Clear the box completely
entry.insert(0, "hello")Put text into the box from your code
show="*"Hide typed characters — use for passwords!

Checkbutton & Radiobutton

Widget

These widgets let users make choices. Checkbuttons are for ticking multiple options (like toppings on a pizza 🍕). Radiobuttons are for picking just ONE from a list (like a size).

Both use special variable objects to remember what the user picked. These are like normal Python variables, but linked to a widget.

BooleanVar()
True or False
Use with Checkbutton
StringVar()
Stores text
Use with Radiobutton
IntVar()
Stores a number
Can also use with Radio
 check_and_radio.py 
import tkinter as tk

def show_order():
    # .get() on a BooleanVar gives True (ticked) or False (not ticked)
    order = f"Size: {size_choice.get()}\n"
    if cheese_var.get():
        order += "+ Extra cheese\n"
    if peppers_var.get():
        order += "+ Peppers\n"
    if olives_var.get():
        order += "+ Olives\n"
    result_label.config(text=order)

root = tk.Tk()
root.title("Pizza Order 🍕")
root.geometry("300x320")

# ── RADIOBUTTONS: pick ONE size ────────────────
tk.Label(root, text="Choose a size:", font=("Helvetica", 11, "bold")).pack(anchor="w", padx=20, pady=(18, 4))

# StringVar stores whichever size the user picks
size_choice = tk.StringVar(value="Medium")  # "Medium" is selected by default

# All three radiobuttons share the SAME variable (size_choice)
# When you click one, the others automatically un-select
tk.Radiobutton(root, text="Small",  variable=size_choice, value="Small").pack(anchor="w", padx=36)
tk.Radiobutton(root, text="Medium", variable=size_choice, value="Medium").pack(anchor="w", padx=36)
tk.Radiobutton(root, text="Large",  variable=size_choice, value="Large").pack(anchor="w", padx=36)

# ── CHECKBUTTONS: tick any extras ──────────────
tk.Label(root, text="Add extras:", font=("Helvetica", 11, "bold")).pack(anchor="w", padx=20, pady=(12, 4))

# Each topping gets its OWN BooleanVar — they work independently
cheese_var  = tk.BooleanVar()
peppers_var = tk.BooleanVar()
olives_var  = tk.BooleanVar()

tk.Checkbutton(root, text="Extra cheese", variable=cheese_var).pack(anchor="w", padx=36)
tk.Checkbutton(root, text="Peppers",      variable=peppers_var).pack(anchor="w", padx=36)
tk.Checkbutton(root, text="Olives",       variable=olives_var).pack(anchor="w", padx=36)

tk.Button(root, text="Show my order", command=show_order).pack(pady=12)

result_label = tk.Label(root, text="", font=("Helvetica", 12), justify="left")
result_label.pack()

root.mainloop()
Part of the codeWhat it does
size_choice = tk.StringVar()Creates a special variable that's "linked" to the radiobuttons. When the user picks a button, this variable updates automatically.
value="Small"This is what gets stored in size_choice when the user clicks that radiobutton.
cheese_var.get()Returns True if the checkbox is ticked, False if not.
anchor="w"Align the widget to the left (West). Without this, pack() centres everything.
padx=36Add 36 pixels of space on the left — this indents the options under the label.

Listbox & Combobox

Widget

A Listbox shows a scrollable list of options. Users can pick one (or multiple). A Combobox is a cleaner dropdown menu — less space, same idea.

 listbox_combobox.py 
import tkinter as tk
from tkinter import ttk  # ttk has extra "themed" widgets like Combobox

def show_choices():
    # curselection() gives the positions (numbers) of selected items
    selected_positions = fruit_list.curselection()

    # Loop through positions to get the actual fruit names
    chosen_fruits = []
    for pos in selected_positions:
        chosen_fruits.append(fruit_list.get(pos))

    country = country_combo.get()  # .get() reads the selected dropdown item

    result_label.config(text=f"Fruits: {chosen_fruits}\nCountry: {country}")

root = tk.Tk()
root.title("Listbox and Combobox")
root.geometry("320x310")

# ── LISTBOX ────────────────────────────────────
tk.Label(root, text="Pick your fruits (click to select):").pack(pady=(18, 5))

# selectmode=tk.MULTIPLE lets the user pick more than one item
fruit_list = tk.Listbox(root, height=5, selectmode=tk.MULTIPLE)
fruit_list.pack()

# Add items to the listbox with .insert()
# tk.END means "add to the end of the list"
fruit_list.insert(tk.END, "Apple")
fruit_list.insert(tk.END, "Banana")
fruit_list.insert(tk.END, "Cherry")
fruit_list.insert(tk.END, "Mango")
fruit_list.insert(tk.END, "Grape")

# ── COMBOBOX ───────────────────────────────────
tk.Label(root, text="Choose your country:").pack(pady=(14, 5))

# values= is the list of options in the dropdown
# state="readonly" means the user can only pick from the list, not type freely
country_combo = ttk.Combobox(root, values=["USA", "UK", "France", "Japan"], state="readonly")
country_combo.current(0)  # current(0) sets the default to the first item (index 0)
country_combo.pack()

tk.Button(root, text="Show choices", command=show_choices).pack(pady=12)

result_label = tk.Label(root, text="", font=("Helvetica", 12), justify="left")
result_label.pack()

root.mainloop()
Part of the codeWhat it does
from tkinter import ttkImports the "themed" part of tkinter. Combobox lives here. Always import this when using Combobox.
fruit_list.curselection()Returns the position numbers of selected items, e.g. (0, 2) if Apple and Cherry are selected.
fruit_list.get(pos)Gets the actual text of the item at that position. get(0) = "Apple", get(1) = "Banana", etc.
selectmode=tk.MULTIPLELets the user hold Ctrl and click to select more than one item at once.
country_combo.current(0)Sets which item is shown by default. 0 = first item in the list.

Organising Widgets with grid()

Layout

So far we've used .pack() which just stacks things on top of each other. grid() is much more powerful — it lets you place widgets in exact rows and columns, like a table.

Think of it like a spreadsheet

Every widget gets a row number and a column number. Row 0 is the top, column 0 is the left:

column=0
column=1
row=0
Label "Name:"
Entry box
row=1
Label "Pass:"
Entry (show="*")
row=2
Button — spans across BOTH columns (columnspan=2)
 grid_layout.py 
import tkinter as tk

root = tk.Tk()
root.title("grid() Layout Demo")
root.geometry("340x180")

# A Frame is like a container — we use it to add padding around everything
frame = tk.Frame(root, padx=20, pady=20)
frame.pack()  # the frame itself uses pack() to fill the window

# Now everything INSIDE the frame uses grid()

# row=0, column=0 → top-left
# sticky="e" → push the label to the right side of its cell (like East on a compass)
tk.Label(frame, text="Username:").grid(row=0, column=0, sticky="e", pady=8)

# row=0, column=1 → top-right
# padx=(8,0) adds 8px of space on the LEFT of this widget only
username_entry = tk.Entry(frame, width=20)
username_entry.grid(row=0, column=1, padx=(8, 0))

tk.Label(frame, text="Password:").grid(row=1, column=0, sticky="e", pady=8)

# show="*" hides what the user types (password style)
password_entry = tk.Entry(frame, width=20, show="*")
password_entry.grid(row=1, column=1, padx=(8, 0))

# columnspan=2 makes this button stretch across BOTH columns
tk.Button(frame, text="Login", width=18).grid(row=2, column=0, columnspan=2, pady=16)

root.mainloop()
Part of the codeWhat it does
tk.Frame(root, padx=20, pady=20)A Frame is an invisible box that holds other widgets. padx/pady add breathing room around the edges.
.grid(row=0, column=0)Place this widget in row 0, column 0 (top-left). Change the numbers to move it.
sticky="e"Push the widget to the right (East) side of its grid cell. Use "w" for left, "ew" to stretch across the full cell width.
columnspan=2Make this widget stretch across 2 columns. Great for buttons at the bottom of a form.
padx=(8, 0)Two numbers: left padding = 8px, right padding = 0px. You can set them separately.
Never mix .pack() and .grid() inside the SAME container. Pick one or the other. It's fine to use pack() for the outer Frame and grid() for everything inside it — just like we did above!

Project 1 — Contact Form

Project

Let's put everything together! This project uses labels, entries, a combobox, a checkbutton, and grid() layout — plus a pop-up message when you submit.

 contact_form.py 
import tkinter as tk
from tkinter import ttk       # for Combobox
from tkinter import messagebox # for pop-up messages

def submit_form():
    # Read all the form fields
    name    = name_entry.get()
    email   = email_entry.get()
    topic   = topic_combo.get()
    message = message_entry.get()
    subscribe = subscribe_var.get()

    # Simple check: make sure name and email aren't empty
    if name == "" or email == "":
        # showwarning() pops up a warning dialog
        messagebox.showwarning("Missing info", "Please fill in your name and email!")
        return  # stop here — don't submit

    # If everything looks good, show a success message
    messagebox.showinfo("Sent! ✅", f"Thanks, {name}!\nWe'll reply to {email}")

# ── WINDOW SETUP ───────────────────────────────
root = tk.Tk()
root.title("Contact Form")
root.geometry("420x340")

# A frame with padding so the form doesn't touch the window edges
frame = tk.Frame(root, padx=28, pady=22)
frame.pack()

# Tell column 1 to stretch when the window resizes
frame.columnconfigure(1, weight=1)

# ── ROW 0: Name ────────────────────────────────
tk.Label(frame, text="Name:").grid(row=0, column=0, sticky="e", pady=7, padx=(0, 10))
name_entry = tk.Entry(frame)
name_entry.grid(row=0, column=1, sticky="ew")  # "ew" = stretch left to right

# ── ROW 1: Email ───────────────────────────────
tk.Label(frame, text="Email:").grid(row=1, column=0, sticky="e", pady=7, padx=(0, 10))
email_entry = tk.Entry(frame)
email_entry.grid(row=1, column=1, sticky="ew")

# ── ROW 2: Topic (dropdown) ────────────────────
tk.Label(frame, text="Topic:").grid(row=2, column=0, sticky="e", pady=7, padx=(0, 10))
topic_combo = ttk.Combobox(frame, values=["General", "Support", "Sales"], state="readonly")
topic_combo.current(0)
topic_combo.grid(row=2, column=1, sticky="ew")

# ── ROW 3: Message ─────────────────────────────
tk.Label(frame, text="Message:").grid(row=3, column=0, sticky="e", pady=7, padx=(0, 10))
message_entry = tk.Entry(frame, width=28)
message_entry.grid(row=3, column=1, sticky="ew")

# ── ROW 4: Newsletter checkbox ─────────────────
subscribe_var = tk.BooleanVar()
tk.Checkbutton(frame, text="Subscribe to newsletter", variable=subscribe_var).grid(
    row=4, column=1, sticky="w", pady=8)

# ── ROW 5: Submit button ───────────────────────
# columnspan=2 so it spans across both label and entry columns
tk.Button(frame, text="Send Message 📨", command=submit_form).grid(
    row=5, column=0, columnspan=2, pady=6)

root.mainloop()

Opening a New Window

Multi-window

What if you want a button to open a second window? In tkinter, extra windows are created using Toplevel.

⚠️
Very important rule! Always use tk.Toplevel(root) for extra windows. NEVER create a second tk.Tk() — that causes bugs and crashes. There should only ever be ONE tk.Tk() in your whole program.
 second_window.py 
import tkinter as tk

def open_second_window():
    # Create a new window — Toplevel(root) links it to the main window
    second_win = tk.Toplevel(root)
    second_win.title("Second Window")
    second_win.geometry("250x130")

    # Add widgets to the second window just like the main one
    tk.Label(second_win, text="I'm a second window! 👋",
             font=("Helvetica", 12)).pack(pady=24)

    # .destroy() closes THIS window when clicked
    tk.Button(second_win, text="Close this window",
              command=second_win.destroy).pack()

# ── MAIN WINDOW ────────────────────────────────
root = tk.Tk()
root.title("Main Window")
root.geometry("300x150")

tk.Label(root, text="This is the main window",
         font=("Helvetica", 12)).pack(pady=28)

tk.Button(root, text="Open a new window",
          command=open_second_window).pack()

root.mainloop()
Part of the codeWhat it does
tk.Toplevel(root)Creates a new window that's linked to the main window. When you close root, this closes too.
second_win.title("...")Set the title of the second window — same as root.title() but for this window.
command=second_win.destroyCloses only the second window. root is not affected. (Again: no brackets after destroy!)
💡
Tip: prevent duplicate windows Clicking the button multiple times opens multiple windows. To prevent this, check if a window already exists: if second_win is None or not second_win.winfo_exists(): before creating a new one.

Project 2 — Two-Window App

Final Project

Our final project! The main window shows a name. A button opens a second window where you can edit that name. When you click Save, the second window closes and the main window updates. This is how real apps pass information between windows.

💬
The trick: callbacks The second window needs to send data back to the first. We do this by passing a function to the second window. When the user clicks Save, the second window calls that function with the new data. This function is called a "callback."
 two_window_app.py 
import tkinter as tk

# ── This function opens the Edit window ────────
def open_edit_window():

    # This inner function runs when the user clicks "Save" in the edit window
    # It takes the new name and updates the main window's label
    def save_name():
        new_name = edit_entry.get()   # read from the edit window's entry box
        name_label.config(text=f"Name: {new_name}")  # update main window label
        edit_window.destroy()         # close the edit window

    # ── Create the Edit window ─────────────────
    edit_window = tk.Toplevel(root)
    edit_window.title("Edit Name")
    edit_window.geometry("280x140")

    # grab_set() makes this window "modal" — the user must deal
    # with it before they can click on the main window again
    edit_window.grab_set()

    tk.Label(edit_window, text="Enter a new name:",
             font=("Helvetica", 11)).pack(pady=(20, 5))

    edit_entry = tk.Entry(edit_window, width=25, font=("Helvetica", 11))

    # Pre-fill the entry with the CURRENT name from the main window
    current_name = name_label.cget("text").replace("Name: ", "")
    edit_entry.insert(0, current_name)
    edit_entry.pack()

    # The Save button calls save_name() — which lives ABOVE in this same function
    tk.Button(edit_window, text="Save ✅", command=save_name).pack(pady=12)

# ── MAIN WINDOW ────────────────────────────────
root = tk.Tk()
root.title("My App")
root.geometry("300x160")

# This label shows the current name — it gets updated when Save is clicked
name_label = tk.Label(root, text="Name: Alice",
                      font=("Helvetica", 15))
name_label.pack(pady=34)

tk.Button(root, text="✏️  Edit name",
          command=open_edit_window).pack()

root.mainloop()
Part of the codeWhat it does
def save_name(): inside def open_edit_window():A function inside a function! save_name() can "see" the variables from open_edit_window() — like edit_entry and edit_window. This is called a closure in Python.
edit_window.grab_set()Makes the edit window "modal" — until the user closes it, the main window is blocked. Like a dialog box that needs attention.
name_label.cget("text").cget() reads a widget's current setting. Here we read the label's text to pre-fill the entry box.
.replace("Name: ", "")Strip the "Name: " prefix so we just get the name itself — e.g. "Alice" instead of "Name: Alice".
